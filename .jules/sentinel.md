## 2024-03-24 - Fix Audit Log Identity Loss via Auth Hydration Race Condition
**Vulnerability:** LGPD audit logs (`src/lib/audit.ts`) were attributing critical clinical data interactions to "system" instead of the authenticated user. This occurred because `auth.currentUser` was read synchronously during rapid UI mutations, frequently evaluating to `null` before Firebase auth state had fully hydrated from IndexedDB/local storage.
**Learning:** In Next.js/React applications with Firebase, relying on the synchronous getter `auth.currentUser` is inherently unsafe for critical security functions like audit logging, especially on initial loads or rapid successive mutations. The authentication state requires an asynchronous cycle to resolve.
**Prevention:** Always wrap `onAuthStateChanged` in a Promise (e.g., `getCurrentUser`) when you need to definitively ascertain the current authenticated user's identity before performing a security-sensitive write or validation, ensuring you await this resolution before proceeding.
