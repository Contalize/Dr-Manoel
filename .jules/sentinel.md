## 2024-04-02 - [MEDIUM] Fix auth race condition in audit logs
**Vulnerability:** Firebase `auth.currentUser` was accessed synchronously when saving audit logs, creating a race condition where the current user ID could resolve to "system" or "anonymous" if the auth state hadn't fully initialized.
**Learning:** This is particularly problematic in modern frameworks like Next.js where page loads or specific component interactions might occur faster than the Firebase Authentication listener can establish the session state on the client. Also, Firebase v9+ provides a built-in `await auth.authStateReady()` method which achieves the exact same result much more cleanly than wrapping `onAuthStateChanged` in a Promise.
**Prevention:** Avoid accessing `auth.currentUser` synchronously. Always use `await auth.authStateReady()` to correctly resolve the authenticated user ID before executing actions that require a valid user identifier, such as submitting documents with strict row-level security or audit logging.
