## 2025-02-17 - [Critical] Missing Row-Level Security for LGPD Compliance
**Vulnerability:** Document creation logic across multiple collections (patients, consultations, evolutions, professionals, prescriptions) omitted the `userId` field. This omission forced Firestore Security Rules to be dangerously permissive (`allow read, write: if request.auth != null;`), effectively bypassing row-level access controls required for LGPD compliance.
**Learning:** Technical debt from rapid feature development led to the omission of the `userId` field upon document creation. The security rules were temporarily relaxed to prevent permission errors, which inadvertently exposed all data to any authenticated user.
**Prevention:** Enforce the presence of `userId` mapping to `auth.currentUser?.uid` on all document creations. Implement strict Firestore rules (`allow create: if request.auth != null && request.resource.data.userId == request.auth.uid;`) to ensure data ownership from the start. Regularly audit document creation endpoints and security rules to maintain strict row-level access controls.
