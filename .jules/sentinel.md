## 2026-04-15 - [CRITICAL] Prevent Race Conditions and Enforce Row-Level Access
 **Vulnerability:** Unauthenticated document creation and missing row-level access control. Creating patient documents without resolving the Firebase authentication state can lead to unhandled exceptions or documents being created without a clear `userId` tie, violating LGPD strict row-level access rules.
 **Learning:** Firebase client libraries load asynchronously. Accessing `auth.currentUser?.uid` immediately during component mount or user action before auth state initialization causes race conditions where `auth.currentUser` is `null`, even if the user is technically logged in from a previous session.
 **Prevention:** Always use `await auth.authStateReady()` before resolving `auth.currentUser` to ensure the auth state is fully loaded. Explicitly verify the existence of `auth.currentUser` before proceeding. Inject `userId: auth.currentUser.uid` in newly created documents to enforce strict row-level access policies (LGPD) without modifying the global `match /{document=**}` firestore rule which breaks system-wide collections.
