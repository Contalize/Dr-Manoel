## 2024-03-12 - Missing userId when creating sensitive records\n**Vulnerability:** New patients and prescriptions were created without attaching the authenticated user ID (`userId`).\n**Learning:** This technical debt left documents vulnerable to overly permissive Firestore rules that allow access to anyone authenticated.\n**Prevention:** Ensure all `addDoc` payloads for sensitive records include `userId: auth.currentUser?.uid` to support row-level access control.
